# OpenHand v0.8.0 ‚Äî Release Notes

_Released: 2026-04-25_

This is the largest single OpenHand release to date. v0.8 lands a real
Model Context Protocol (MCP) integration, a layered Sandbox v2 with
opt-in CPU / memory / network guards, a fresh in-tree plugin, and a
deep chaos-test pass that hardened the runtime against malformed
plugins, network failures, oversized payloads, and SSE disconnects.

## Highlights

### MCP (Model Context Protocol) integration

- New `@openhand/mcp` package ‚Äî spawn-and-talk style. `MCPClient`
  starts an MCP server as a child process, performs the JSON-RPC 2.0
  `initialize` handshake over line-delimited JSON on stdio, and
  exposes `listTools()` / `callTool(name, args)`.
- `bridgeMcpTools(client)` returns a `Map<string, Tool>` ready to
  merge into `createTools()` ‚Äî every MCP tool becomes a first-class
  OpenHand tool.
- Zero runtime deps. ~120 lines of JSON-RPC framing in
  `packages/mcp/src/jsonrpc.ts`. No `@modelcontextprotocol/sdk`.
- v0.8 covers `initialize` + `tools/list` + `tools/call` over stdio.
  Resources, prompts, sampling, and HTTP/SSE transport are out of
  scope for this release.
- `plugins/mcp-bridge/` ships a single-dispatcher plugin
  (`mcp_list_tools`, `mcp_call`) for runtime-selectable MCP servers.
- Runnable demo: `npx tsx examples/mcp-demo.ts`.
- Recipe: `cookbook/08-mcp-integration.md`.

### Sandbox v2 ‚Äî opt-in defense in depth

`packages/sandbox/src/v2.ts` adds three composable guards on top of v1:

- `wrapWithCpuLimit(cmd, args, ms)` ‚Äî uses `timeout(1)` / `gtimeout`
  from coreutils when on `PATH`. v1's wallclock SIGKILL escalation
  still applies otherwise.
- `wrapWithMemoryLimit(cmd, args, mb)` ‚Äî prefers `prlimit --as=`,
  falls back to `sh -c 'ulimit -v <kb>; exec "$0" "$@"'`. Positional
  args round-trip verbatim; the adversarial
  `"/tmp/oh; rm -rf /"` case is in the test suite.
- `createNetGuardedFetch()` ‚Äî fetch wrapper that rejects every call
  with `NetworkBlockedError` when `NET=none` is in the env. Strict
  equality only ‚Äî `NET=0` / `false` / `off` do NOT block. Snapshot at
  construction so flipping the env mid-process can't create racy
  half-isolation.

Design rationale and portability matrix in `docs/SANDBOX_v2.md`.

### Fortune-cookie plugin

`plugins/fortune-cookie/` ‚Äî a new in-tree plugin that ships a tiny
Markov-chain fortune generator with a deterministic seed mode for
tests. It's the smallest end-to-end example we ship: 325-line single
file, no external deps, six unit tests. Recommended starting point
for anyone writing their first plugin.

### Chaos test suite

`tests/chaos/` ‚Äî five new test files covering failure modes the unit
suite can't reach:

- `malformed-plugin.test.ts` ‚Äî broken manifests, infinite-loop
  init, plugins that throw synchronously / asynchronously.
- `network-failure.test.ts` ‚Äî provider client behavior on DNS
  failure, ECONNRESET, half-closed sockets, slow drip.
- `large-payload.test.ts` ‚Äî 10 MB tool inputs / outputs, deeply
  nested JSON, NUL-byte handling.
- `sse-disconnect.test.ts` ‚Äî `Last-Event-ID` replay across mid-stream
  client disconnect.
- `random-cli-input.test.ts` ‚Äî fuzz-style random byte feed at the
  REPL prompt.

Run with `npm run test:chaos`.

### Plus

- New CLI commands: `openhand audit` (vuln + plugin posture report)
  and `openhand doctor` (env/runtime self-check).
- `@openhand/core` telemetry module ‚Äî opt-in metrics emitter,
  off by default, no PII.
- Real-provider smoke tests under `tests/real/` for OpenAI,
  Anthropic, and Ollama. Gated on credentials / a reachable daemon
  and skipped otherwise. New `npm run test:real` script ‚Äî default
  `npm test` doesn't pick these up.
- Three new cookbook recipes (multi-agent orchestration, streaming
  tool use, MCP integration) and a `quick-plugin` walkthrough.
- Plugin marketplace doc at `docs/PLUGIN_MARKETPLACE.md`.
- Observability guide at `docs/OBSERVABILITY.md`.

## Upgrading

`@openhand/core`, `@openhand/llm`, `@openhand/sandbox`,
`@openhand/tools`, `@openhand/mcp`, `@openhand/cli`,
`@openhand/server`, and `@openhand/web` all bump to **0.8.0**.

No breaking API changes. Sandbox v2 is purely additive ‚Äî v1 helpers
remain in place. MCP support is in a new package; existing imports
are unchanged.

```bash
npm install
npm run typecheck
npm test
```
